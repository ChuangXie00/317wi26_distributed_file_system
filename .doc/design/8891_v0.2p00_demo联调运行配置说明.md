# 8891_v0.2p00 Demo 联调运行配置说明

更新时间：2026-03-13（America/Los_Angeles）

## 1) 目标

本文件用于 Commit 10 联调验收阶段，统一 Demo Frontend / Demo Backend / 集群容器的启动配置，避免“端口通但数据源错位”。

## 2) 固定端口与访问路径

- Frontend（Vite）：`http://127.0.0.1:5173`
- Demo Backend（FastAPI）：`http://127.0.0.1:18080`
- Meta Entry（Nginx）：`http://127.0.0.1:8000`
- Demo API Base：`http://127.0.0.1:18080/api/demo`

## 3) Demo Backend 启动配置

在仓库根目录执行：

```powershell
$env:DEMO_META_ENTRY_BASE_URL = "http://127.0.0.1:8000"
$env:DEMO_COMPOSE_FILE = "deploy/docker-compose.yml"
$env:DEMO_COMPOSE_TIMEOUT_SEC = "20"

cd services/demo_backend
..\..\.venv\Scripts\python.exe -m uvicorn app:app --host 127.0.0.1 --port 18080
```

变量说明：

- `DEMO_META_ENTRY_BASE_URL`：将聚合上游从容器域名改为宿主机可访问地址。
- `DEMO_COMPOSE_FILE`：固定动作执行 compose 文件路径，避免 cwd 差异。
- `DEMO_COMPOSE_TIMEOUT_SEC`：动作接口超时阈值（秒）。

## 4) Frontend 启动配置

在仓库根目录执行：

```powershell
$env:VITE_DEMO_API_BASE_URL = "http://127.0.0.1:18080/api/demo"
cd frontend
npm.cmd run dev -- --host
```

## 5) 联调前健康检查

```powershell
docker ps --format "table {{.Names}}\t{{.Status}}"
curl.exe -sS http://127.0.0.1:18080/api/demo/health
curl.exe -sS http://127.0.0.1:18080/api/demo/state
```

通过标准：

- Docker 中 `meta-entry/meta-01/meta-02/meta-03/storage-01/02/03/postgres` 均为 `Up`。
- `/health` 返回 `ok=true`。
- `/state` 返回 `ok=true`，`membership.summary.total=6`。

## 6) 推荐演练顺序（15 分钟路径）

1. 观察基线：leader、entry、membership、lamport。
2. 执行 `stop` 当前 leader，观察新 leader 当选。
3. 确认 `entry.active_leader_id` 已切到新 leader。
4. 执行 `start` 原 leader，确认其以 follower 回归。
5. 校验 `/events` 与 `/metrics`：`leader.switched`、`recovery.completed`、`failover_recovery_seconds`。
